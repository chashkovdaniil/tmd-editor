# AGENTS.md

## Проект

`tmd-editor` - плагин Obsidian для просмотра и редактирования файлов тренировок
с расширением `.tmd`. Формат файла похож на Markdown: опциональный заголовок
тренировки пишется как `[[...]]`, упражнение начинается с `###`, под ним может
быть заметка и таблица подходов.

## Структура

- `src/main.ts` - точка входа плагина. Регистрирует кастомный view для `.tmd`,
  пункты файлового меню Obsidian и вкладку настроек.
- `src/TmdView.tsx` - `FileView`, который читает текущий `TFile`, парсит
  содержимое, рендерит React-редактор и автосохраняет изменения через vault.
- `src/components/TmdEditor.tsx` - основной React-компонент редактора:
  редактирование таблиц, добавление/удаление подходов и упражнений,
  переименование файла.
- `src/TmdParser.ts` - типы `TmdFile`/`TmdExercise`, парсер и сериализатор
  `.tmd`-контента.
- `types/obsidian.d.ts` - локальные TypeScript-типы для используемых API
  Obsidian.
- `manifest.json` - метаданные Obsidian-плагина.
- `.github/workflows/ci.yml` - GitHub Actions: build на PR и release на push
  в `main`.
- `dist/` - результат сборки; создается командой `npm run build`.
- `Instruction.md`, `Startup.md`, `memorybank.md` - проектный контекст и
  заметки по запуску.

## Команды

- Установка зависимостей: `npm ci`.
- Сборка и проверка TypeScript/esbuild: `npm run build`.
- CI-сборка для автоматизации: `npm run ci:build`.
- GitHub Actions использует `npm run ci:build`; релиз создается после push в
  `main` с артефактами `dist/main.js`, `dist/manifest.json` и zip-архивом.
- Отдельных тестов или lint-скриптов в `package.json` нет.

## Важные паттерны

- Сохраняйте связку `parseTmd`/`serializeTmd`: изменения формата `.tmd`
  должны поддерживаться в обе стороны.
- В `TmdEditor` пользовательские изменения должны выставлять `dirty=true`;
  автосохранение запускается эффектом только после реального изменения.
- При открытии другого файла состояние редактора сбрасывается из `initialTmd`.
- `TmdView` отвечает за работу с Obsidian vault и уведомления, а `TmdEditor`
  должен оставаться UI-компонентом, принимающим `tmd`, `file`, `app` и `refTmd`.
- Для файлового меню и настроек используйте API Obsidian в `src/main.ts`.

## Проверка изменений

После изменения исходников запускайте `npm run build` или полный CI-вход
`npm run ci:build`, если нужно проверить установку зависимостей с нуля. Для
документации достаточно проверить отсутствие лишних пробелов в конце строк и
наличие новой строки в конце файла.
